Read the `tuist` documentation before proceeding with it.

2. `tasks`: This folder contains only markdown files. Similar to GitHub issues or project management software, we track our ongoing issues in plain markdown. Each issue (feature, fix, improvement) is documented in its own markdown file. When a markdown issue is completed, move it to `tasks/archive`. `tasks/archive` only exists as a historical reference.
